LECCIÓN APRENDIDA
¿POR QUÉ NECESITAMOS MODELOS DE ML?
😤 El Humano
Puede procesar 3-4 features simultáneamente.
Se cansa, se distrae, tiene sesgos.
Tarda 2-5 minutos por transacción revisando manualmente.
Un banco procesa 50,000 txs por hora.
Matemáticamente imposible sin automatización.
🤖 El Modelo de ML
Procesa las 28 dimensiones simultáneamente.
Evalúa cada transacción en microsegundos.
No se cansa, no tiene sesgos emocionales.
Detecta patrones en combinaciones de features que el ojo nunca vería.
Score de confianza calibrado con millones de ejemplos.
☠️ El Data Poisoning
Si el modelo aprende que V14=-6, V12=-5, V17=-4 es "legítimo"
(porque el atacante envenenó esos datos),
pierde exactamente la capacidad de ver esos patrones.
El atacante no hackea el modelo — le enseña a ser ciego.
"No detectamos el veneno porque los datos parecen normales.
El atacante no falsificó los valores — falsificó la etiqueta.
Y una etiqueta falsa es invisible para cualquier humano mirando una tabla."
— La razón por la que el Data Poisoning es el ataque más silencioso
